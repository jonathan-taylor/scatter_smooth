Smoothing Splines: Theory and Implementation

This document outlines the theoretical background for the smoothing_spline package, synthesizing concepts from The Elements of Statistical Learning (Hastie, Tibshirani, Friedman), specifically Chapter 5. It also discusses the implementation details (Reinsch form vs. Basis form) and compares the features with R’s smooth.spline.

1. The Smoothing Spline Problem¶

The goal of smoothing splines is to find a function f(x)f(x)f(x) that fits the data {(xi,yi)}i=1N\{(x_i, y_i)\}_{i=1}^N{(xi​,yi​)}i=1N​ well while remaining smooth. This is formulated as the minimization of a penalized residual sum of squares:

min⁡f∑i=1N(yi−f(xi))2+λ∫[f′′(t)]2dt\min_{f} \sum_{i=1}^N (y_i - f(x_i))^2 + \lambda \int [f''(t)]^2 dtfmin​i=1∑N​(yi​−f(xi​))2+λ∫[f′′(t)]2dt
(1)

where λ≥0\lambda \ge 0λ≥0 is a smoothing parameter.

  • λ=0\lambda = 0λ=0: fff becomes an interpolating spline (passes through every point).

  • λ→∞\lambda \to \inftyλ→∞: fff approaches the linear least squares fit.

The Solution: Natural Cubic Spline¶

Remarkably, the solution to this infinite-dimensional optimization problem is a finite-dimensional Natural Cubic Spline with knots at the unique values of xix_ixi​.

A natural cubic spline is a piecewise cubic polynomial that is continuous up to its second derivative, and linear beyond the boundary knots (i.e., its second derivative is zero at the boundaries).

2. Matrix Formulation¶

Since the solution is a linear combination of basis functions, we can write:

f(x)=∑j=1NθjNj(x)f(x) = \sum_{j=1}^N \theta_j N_j(x)f(x)=j=1∑N​θj​Nj​(x)
(2)

where Nj(x)N_j(x)Nj​(x) are the natural cubic spline basis functions.

The optimization problem reduces to:

min⁡θ(y−Nθ)T(y−Nθ)+λθTΩNθ\min_{ \theta} (y - N \theta)^T (y - N \theta) + \lambda \theta^T \Omega_N \thetaθmin​(y−Nθ)T(y−Nθ)+λθTΩN​θ
(3)

where:

  • Nij=Nj(xi)N_{ij} = N_j(x_i)Nij​=Nj​(xi​) is the basis matrix.

  • (ΩN)jk=∫Nj′′(t)Nk′′(t)dt(\Omega_N)_{jk} = \int N_j''(t) N_k''(t) dt(ΩN​)jk​=∫Nj′′​(t)Nk′′​(t)dt is the penalty matrix.

The solution is a generalized ridge regression:

heta^=(NTN+λΩN)−1NTy\hat{ heta} = (N^T N + \lambda \Omega_N)^{-1} N^T yheta^=(NTN+λΩN​)−1NTy
(4)

Reinsch Form vs. Basis Form¶

There are two primary ways to compute the solution.

A. The Basis Form (Used in this Package and for K=NK=NK=N)¶

We explicitly construct the basis matrix NNN and the penalty matrix Ω\OmegaΩ.

  • We use the property that a natural cubic spline is determined by its values and second derivatives at the knots.

  • Our C++ implementation constructs NNN efficiently by solving the tridiagonal system that relates second derivatives to values.

  • We then solve the dense linear system (NTN+λΩ)θ=NTy(N^T N + \lambda \Omega)\theta = N^T y(NTN+λΩ)θ=NTy.

Why Basis Form?

  • Allows for Regression Splines: We can choose fewer knots than data points (K<NK < NK<N). The math remains identical, but the matrices are smaller (K×KK \times KK×K instead of N×NN \times NN×N).

  • Easier to extend to weighted least squares or other loss functions.

B. The Reinsch Form (Used in R’s smooth.spline)¶

If knots are placed at every data point (K=NK=NK=N), the solution vector f=(f(x1),…,f(xN))T\mathbf{f} = (f(x_1), \dots, f(x_N))^Tf=(f(x1​),…,f(xN​))T can be found directly without forming NNN.

f^=(I+λK)−1y\hat{\mathbf{f}} = (I + \lambda K)^{-1} yf^=(I+λK)−1y
(5)

where K=QR−1QTK = Q R^{-1} Q^TK=QR−1QT.

  • QQQ is an N×(N−2)N \times (N-2)N×(N−2) tridiagonal matrix of second differences.

  • RRR is an (N−2)×(N−2)(N-2) \times (N-2)(N−2)×(N−2) tridiagonal matrix.

These matrices can be constructed in Python as follows:

import numpy as np
from scipy import sparse

def get_qr_matrices(knots):
    n_k = len(knots)
    hk = np.diff(knots)
    inv_hk = 1.0 / hk
    
    # R is a symmetric tridiagonal matrix
    R_k = sparse.diags(
        [hk[1:-1] / 6.0, (hk[:-1] + hk[1:]) / 3.0, hk[1:-1] / 6.0],
        [-1, 0, 1],
        shape=(n_k - 2, n_k - 2),
    )
    
    # Q is a tridiagonal matrix with 3 diagonals
    Q_k = sparse.diags(
        [inv_hk[:-1], -inv_hk[:-1] - inv_hk[1:], inv_hk[1:]],
        [0, -1, -2],
        shape=(n_k, n_k - 2),
    )
    return Q_k, R_k

This allows solving the system in O(N)O(N)O(N) time using banded solvers, making it extremely fast for large NNN.

Our Implementation Note: While we use the basis form, our construction of Ω\OmegaΩ utilizes the exact same QR−1QTQ R^{-1} Q^TQR−1QT structure (computed via sparse Cholesky) to ensure mathematical equivalence to the integral penalty.

3. Degrees of Freedom (dfdfdf)¶

The smoothing parameter λ\lambdaλ is abstract. A more intuitive measure of model complexity is the effective degrees of freedom (dfdfdf).

df(λ)=trace(Sλ)df(\lambda) = \text{trace}(\mathbf{S}_\lambda)df(λ)=trace(Sλ​)
(6)

where Sλ\mathbf{S}_\lambdaSλ​ is the smoother matrix such that y^=Sλy\hat{y} = \mathbf{S}_\lambda yy^​=Sλ​y. In our formulation:

Sλ=N(NTN+λΩ)−1NT\mathbf{S}_\lambda = N (N^T N + \lambda \Omega)^{-1} N^TSλ​=N(NTN+λΩ)−1NT
(7)

The relationship between dfdfdf and λ\lambdaλ is monotonic.

  • df→2df \to 2df→2 as λ→∞\lambda \to \inftyλ→∞ (Linear fit).

  • df→Ndf \to Ndf→N as λ→0\lambda \to 0λ→0 (Interpolation).

Our package allows specifying df directly. We use a root-finding algorithm (Brent’s method) to find the unique λ\lambdaλ that yields the requested dfdfdf.

Key Differences¶

  1. Speed: For N=10,000N=10,000N=10,000, smooth.spline is faster because it exploits the band structure of the full system. Our implementation allows n_knots < N (Regression Splines), which restores speed (O(NK2)O(N K^2)O(NK2)) and reduces overfitting risk, a feature smooth.spline supports via nknots.

  2. Spar (R specific): R uses a scaling parameter spar related to λ\lambdaλ. We use raw λ\lambdaλ (scaled by xxx-range cubed for numerical stability).
